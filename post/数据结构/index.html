switch-case语句

static的作用

  • 静态函数：只能在定义它的文件中使用
  • 静态变量：初始值为0，在程序运行期间一直存在
  • 静态代码块：只有在函数第一次被调用时才会运行
  • 缺省情况下全局可见

优先级

  • .的优先级高于*和->
  • []的优先级高于*
  • ()的优先级高于*
  • !=和==的优先级高于位运算
  • 算术运算的优先级高于位运算
  • 逗号在所有运算符中优先级最低

const和*的不同位置表示的含义

  • const在*之前(const int *a; int const *b)，指针所指对象只读
  • const在*之后(int *const c;)，指针只读
  • const在*之前和之后都有(const int * const d; int const *const e;)，指针和指针所指对象只读

传值、传址、传引用

  • 传值：function(x)/void function(int x)
  • 传址：function(&x)/void function(int *x)
  • 传引用：function(x)/void function(int &x)

基本概念和术语

  • 数据：对客观事物的符号表示，在计算机科学中是指所有能输入到计算机中并被计算机程序处理的符号的总称。
  • 数据元素：数据的基本单位，是数据结合中的一个个体，一个数据元素可由若干个数据项组成。
  • 数据对象：性质相同的数据元素的集合，是数据的一个子集。
  • 数据结构：相互之间存在一种或多种特定关系的数据元素的集合。
  • 数据结构：(D,S,Op)。其中D位数据元素的集合；S为D上的关系Op为定义在D上的运算

线性表

形式定义

  • 由n(n>=0)个数据元素组成的有序序列，n为表长，n为0时，是空表
  • Linear_list = (D, S)
  • 数据元素间是线性关系，数据元素在表中的位置只取决于其序号
  • 存在唯一的“第一个”和“最后一个”数据元素
  • 除第一个元素外，每一个元素都有一个直接前驱元素，除最后一个元素外，每个元素都有一个直接后继元素

顺序表（向量）

  • malloc和free，检查是否malloc成功

优点

  • 逻辑相邻、物理相邻
  • 可随机存取任一元素
  • 存储空间使用紧凑

缺点

  • 插入、删除操作需移动大量元素
  • 按最大空间分配，利用不充分
  • 容量扩充困难

判空

  • H为头指针，有头结点
  • 不能用H->next==H->prev来判空，若仅有一个数据元素也成立
  • 可用H->next==H或H->prev==H来判断

静态链表

  • 用数组模拟链表，逻辑上看像链表，物理上看像数组

线性表的应用

  • 一元n次多项式相加

栈和队列

栈的定义

  • 特殊的线性表：顺序方式或链式方式存储
  • 插入和删除操作只能在表尾进行
  • 具有后进先出的特点

顺序栈

  • 数组（栈空间）+ 栈顶指示
  • 栈空条件S.top == -1
  • 栈满条件S.top == max - 1
  • *n个元素依次入栈，可得到多少个合法的出栈序列：(2n)!/[(n+1)!n!]

链式栈

  • 链式栈无栈满问题，空间可扩充
  • 插入与删除仅在栈顶处执行
  • 链式栈的栈顶在链头
  • 适合于多栈操作

栈的应用

递归问题（汉诺塔问题）

  • 空间复杂度O(n)最大空间
  • 时间复杂度O(2n)

消除尾递归

  • goto回到程序最开头
  • 改为循环

整形数简单表达式求值

队列的定义

  • 队列是一种特殊的线性表，限定插入和删除操作分别在表的两段进行，先进先出

优先级队列

  • 内部元素按照权值排序，权值较高者排在最前优先出队列

链队列

  • 类型定义：队头指针+队尾指针
  • 存在问题：假上溢

循环队列

  • 设循环队列中数组的下标范围是1~n，其头尾指针分别是f和r，则其元素个数为 (r-f+n) mod n

判断队满

  • 牺牲一个空间
  • 统计元素个数

双端队列

  • 限定插入和删除操作在线性表的两端进行

字符串与模式匹配

基本概念

  • 串：由另个或多个字符组成的优先序列，串是特殊的线性表，数据元素是单个字符
  • 串长：串中的字符的数目
  • 子串和主串：串中任意个连续的字符组成的子序列称为该串的子串，包含子串的串相应的称为主串
  • 串相等：长度相等，各个对应位置的字符都相等
  • 空串和空格串：空串中不包含任何字符，空格串由一个或多个空格组成

串的存储结构

静态存储结构

  • 非紧缩格式：串空间+串长
  • 紧缩格式：串空间+串长

动态存储结构

  • 块链存储方式：字符串块链表的头指针+串长
  • 堆结构存储方式：系统分配一个大容量连续存储空间作为串值的可利用空间。每个串使用时动态申请一组地址连续的存储单元

串的模式匹配

  • 在主串中寻找子串的位置
  • 蛮力搜索，时间复杂度O(n2)
  • KMP算法，时间复杂度O(m+n)

KMP算法

  • 引入next数组，当第k个元素比较出错时，再与next[k]个元素进行比较

数组和广义表

数组

  • 任何数组A都可以看作一个线性表
  • n维数组可以看成m个n-1维数组
  • 数组作为函数形参时必须提供除最左边一维外所有维的长度，把实参限制在除最左边一维外的所有维都必须与形参匹配的数组
  • Pascal、C语言按行存储，Fortraon语言按列存储

矩阵的压缩存储

  • 对称矩阵 ak = i*(i - 1)/2 + j
  • 三角矩阵，对角线以下（或以上）的数据元素（不包括对角线）全部为常数C 重复元素C共享一个元素存储空间
  • *带状矩阵 以对角线顺序存储，共nL个元素，存储地址计算：k=(i-1)L+1+(j-1)
  • 稀疏矩阵 顺序存储：三元组表；链式存储：十字（正交）链表

广义表

概念

  • 广义表是由零个或多个原子或子表组成的有限序列，可以记作：LS=(d1, d2,…… dn)
  • 逻辑定义 Lists = (D, R)
  • 与线性表的关系：广义表中的元素全部为原子时即为线性表，线性表时广义表的特例，广义表是线性表的推广
  • 表的长度：表中的元素（第一层）的个数
  • 表的深度：表中元素的最深嵌套层数
  • 表头：表中的第一个元素
  • 表尾：除第一个元素外，剩余元素构成的广义表

广义表的存储方式

  • 头尾链表形式 原子结点（0，data）、表结点（1，hp，tp）
  • 扩展的线性链表形式 表结点（1，hp，tp）、原子结点（0，data，tp）

树与二叉树

树

  • 重要的非线性数据结构，是以分支关系定义的层次结构
  • 树是由n(n>=0)个结点组成的有限集合T
  • 除根节点外，每个节点都仅有一个前驱结点
  • 结点的度：子树的数目（分支的数目）
  • 叶子结点：度为0的结点
  • 分支（非终端）结点：度不为0的结点
  • 树的度：树的各结点度的最大值
  • 分支数：n-1条边（结点数为n）
  • 在树中任意两结点之间连线，构成图状结构
  • 内部结点：除根节点之外的分支结点
  • 孩子结点
  • 双亲结点
  • 兄弟结点
  • 结点的祖先：从根到该结点所经分支上的所有结点
  • 结点的子孙：该结点为根的子树中的任何一个结点
  • 结点的层次：表示该结点在树中的相对位置，根为第一层，其他的结点依次下推，若某结点在L层上，则其孩子在L+1层上
  • 堂兄弟：双亲在同一层的结点互为堂兄弟
  • 树的高度（深度）：树中结点的最大层次
  • 路径长度：从树中某结点Ni出发，能够“自上而下”的穿过某些结点，到达另一个结点Nj，则称Ni到Nj存在一条路径，路径长度等于这两个结点之间的分支数
  • 树的路径长度：根到每个结点的路径长度之和
  • 森林：是m棵互不相交的树的集合

非空树

  • 有一个称之为根的结点
  • 除根以外的其余结点被分成m(0<=m<n)个互不相交的集合，其中每一个结合是一棵树，且称为根的子树

二叉树

定义

  • 二叉树是n(n>=0)个结点的有限集合，它成为空树(n=0)，或由一个根节点和两颗互不相交的左子树和右子树的二叉树组成
  • 定义是递归的
  • 0<=结点的度<=2（0、1、2）
  • 是有序树
  • 满二叉树 每一层上的结点数都是最大结点数
  • 完全二叉树 只有最下面两层结点的度可以小于2，而最下一层的叶子结点集中于左边的若干位置上

性质

  • 二叉树第i层上至多有2i-1个结点
  • 深度为k的二叉树至多有2k-1个结点
  • 对任何一棵二叉树，如果其终端结点数为n0，度为2的结点数为n2，则n0=n2+1
  • 具有n个结点的完全二叉树的深度为log2n+1（向下取整）
  • 一棵具有n个结点的完全二叉树（顺序二叉树）对其结点编号1～n，则对于任意一结点i，有其左右孩子结点的编号为2i-1和2i+1

二叉树的存储结构

  • 顺序存储结构
  • 三元组(data,lc,rc)
  • 存储双亲(data,parent)
  • 二叉链表(lc,data,rc)
  • 三叉链表(lc,data,parent,rc)

二叉树的遍历

  • 目的：非线性结构->线性结构
  • DLR 先序遍历
  • LDR 中序遍历
  • LRD 后序遍历
  • 层序遍历
  • 先序遍历与中序遍历相当于入栈和出栈操作
  • 中序遍历和后序遍历相当于入栈和出栈操作
  • 中序+先序/后序可推知树的结构
  • 先序+后序遍历不可推知树的结构
  • 层序+中序可推知树的结构
  • 层序+先序/后序可推知树的结构

特殊二叉树

  • 先序序列与后序序列相同：空树/只有根节点的二叉树
  • 中序序列与后序序列相同：空树/任一结点至多只有左子树的二叉树
  • 先序序列与中序序列相同：空树/任一结点至多只有右子树的二叉树
  • 中序序列与层序遍历序列相同：空树/任一结点至多只有右子树的二叉树

线索二叉树

  • 中序线索化
  • (lc, Itag, data, rtag, rc)
  • 先序线索二叉树：最后一个元素即最右下的叶子结点右指针一定为空；第一个元素若没有左子树则左指针为空，若有左孩子，则不为空->有一个或两个空指针
  • 中序线索二叉树：第一个元素没有前驱，最后一个元素没有后继，一定有两个空指针
  • 后序线索二叉树：最后一个元素即根节点的右指针不一定为空；第一个元素即最左下的元素的左指针一定为空->有一个或两个空指针

哈夫曼树

  • 最优二叉树（带权路径长度之和WPL最小的二叉树）
  • 树的路径长度：从树根到每一个结点的路径上的分支数
  • 带权路径长度：结点的路径长度与该结点的权之和
  • 树的带权路径长度：树中所有叶子结点的带权路径长度之和

哈夫曼编码

  • 不等长编码
  • 前缀编码
  • 哈夫曼编码树中没有度为1的结点，若叶子结点的个数为n，则哈夫曼编码树的结点总数为2n-1

森林与二叉树的转换

  • 森林->二叉树：在兄弟之间加一条线
  • 对每一结点，去掉它与孩子的连线（最左孩子除外）
  • 以根为轴心，对整棵树旋转45o

森林的遍历

  • 森林转化成二叉树：(1)每一棵树转换成二叉树。(2)从最后一棵树开始，把后一棵树作为前一棵树根的右子
  • 先序遍历，对应二叉树的先序遍历
  • 中序遍历：对应二叉树的中序遍历

有向图强连通分量的求法

有向无环图的应用

AOV网络

  • 顶点为活动，有向边的指向表示活动执行的次序

AOE网路

  • 顶点为事件，单位是时刻。有向边定义为活动，它的权值定义为活动进行所需的时间

拓扑排序

  • ->拓扑序列（不唯一），AOV网的性质
  • 无前驱顶点优先算法
  • 无后继顶点优先算法 可用深度优先遍历：无法判断是否有环，适用于有向无环图

关键路径

  • AOE网的性质
  • 源点
  • 汇点
  • 关键路径
  • 关键活动
  • e(i)活动ai的最早开始时间，l(i)活动ai的最迟开始时间，ve(j)事件vj的最早发生时间，vl(j)事件vj的最迟发生时间。e(i) = ve(j) l(i) = vl(k) - dut(j, k)
  • 求事件vj的最早发生时间ve(j)，从前往后，在求拓扑排序的过程中实现
  • 求事件vj的最迟发生时间vl(j)，从后往前，逆拓扑排序
  • 求活动ai的最早开始时间e(i)和最迟开始时间l(i)，确定关键活动e(i)=l(i) 根据ve和vl求

Dijkstra算法
  • 最短路径性质：三角不等式性质，最短路径的子路径也是最短路径
  • 最短路径的松弛操作，初始值：P(v)是v可能的最短路径上的前驱结点，D(s)=0, D(v)=∞，P(v) = NULL -> 松弛过程
  • 已知图G,求从图中一个顶点到其他顶点的最段路径，一般使用的算法是Dikstra算法
  • 数据结构：邻接矩阵w[n][n]；一维数组S[1..n]：记录相应顶点是否已被确定最段距离；一维数组D[1..n]：记录源点到相应顶点路径长度；一维数组P[1..n]：记录相应顶点的双亲顶点
  • 时间复杂度O(n2)
  • 适合稠密图：与边的数目无关

弗洛伊德算法

  • 存储结构是邻接矩阵
  • 利用二维数组记录当前vi到vj的最短路径长度
  • 集合S记录当前允许的中间节点
  • 依次向S中加入v1、v2、…vn，每加入一个顶点，对A[i, j]进行一次修正
  • 时间复杂度O(n3)

顺序表的查找

  • 设置监视哨，提高算法效率
  • 改进措施：非等概率查找时，可将查找概率高的记录尽量排在表前部

二分查找（折半查找）

静态树表的查找

  • 以概率高的结点为根，二分查找
  • 静态最优查找树：PH值最小
  • 次优查找树：PH值近似为最小

构造次优查找树

  • 按关键字升序排列，选取记录ri作为根节点，计算左右子树的PH之差，找出最小的PH之差对应的i，若相邻关键字的权值大，则确定为权值大的关键字对应的i n
  • 左右子树也为静态最优查找树

分块查找

  • 构造索引表
  • 折半或顺序查找索引表，确定所在块
  • 在已确定的快中顺序查找/折半查找
  • 分块查找效率介于顺序查找和折半查找之间

二叉排序树BST

  • 左子树<根节点<右子树
  • 新插入的结点总是叶子结点
  • 生成：从空树开始进行插入操作
  • 删除中间节点后，以左子树的最大值或右子树的最小值填充

平衡二叉树AVL（避免二叉排序树退化）

  • 是二叉排序树
  • 左、右子树的高度之差的绝对值不超过1
  • 左右子树本身又是一棵平衡二叉树
  • 二叉树上结点的平衡因子：该结点的左子树的高度减去右子树的高度

平衡二叉树的调整

  • LL型：新插入的结点在危机结点的左子的左子树上->一次顺时针旋转
  • LR型：新插入的结点在危机结点的左子的右子树上->一次逆时针旋转+一次顺时针旋转
  • RR型：新插入的结点在危机结点的右子的右子树上->一次逆时针旋转
  • RL型：新插入的结点在危机结点的右子的左子树上->一次顺时针旋转+一次逆时针旋转
  • AVL树是接近最优的，其平均查找长度与log2n同数量级

B+树与B-树

B-树

  • 多路平衡查找树，应用于文件系统
  • m阶B-树，树中每节点最多有m棵子树，若根节点不是叶子结点，则最少有两棵子树，除根之外的所有非终端节点最少m/2向上取整棵子树，所有非终端节点包含信息数据，所有叶子结点在同一层上且不带信息
  • B-树插入过程，若结点数过多，必须分裂成两个结点
  • B-树删除过程：不动/借/并

B+树

  • B-树的变形树，应用于索引顺序文件系统
  • 有n棵子树的结点中有n个关键字，非叶子结点可以看成是索引部分，所有叶子结点中包含了全部关键字的信息以及指向这些关键字记录的指针，且叶子结点以关键字大小自小至大顺序链接

哈希表

  • 在记录的存储地址和它的关键字之间建立一个确定的对应关系
  • 哈希表
  • 哈希函数
  • 冲突
  • 同义词
  • 哈希（散列）地址
  • 装填因子n/m

哈希函数的基本构造方法

  • 直接定址法
  • 数字分析法/基数转换法：取关键字在某种进制下的若干数位作为哈希地址
  • 平方取中法：关键字平方后取中间几位作为地址
  • 随机数法：较适于关键字长度不等时的情况
  • 折叠法：将关键字分割成位数相同的几部分，然后取这几部分的叠加和作为哈希地址
  • 除留余数法 p最好是素数

处理冲突的常用方法

  • 开放定址法：线性探测再散列、二次探测再散列、为随机探测再散列、双重散列函数探查法
  • 再哈希法
  • 链地址法：为每个哈希地址建立一个单链表
  • 建立公共溢出区

排序

插入排序

普通插入排序（稳定）

  • 算法改进：折半插入排序（稳定、二路插入排序（稳定）

希尔排序（不稳定）

  • 选定一个记录下标的增量d，将整个记录序列按增量d从第一个记录开始划分为若干组，对每组使用直接插入排序的方法，然后减小增量d，不断重复上述过程，如此下去，直到d=1
  • 最后一个增量之必为1
  • 避免增量序列中的值有公因子
  • 实验结果：n比较大时，比较和移动次数约在n1.25到1.6n1.25
  • 原地排序

交换排序

冒泡排序（稳定）

  • 时间复杂度O(n2)
  • 辅助空间复杂度O(1)
  • 算法改进：每趟排序中，记录最后一次发生交换的位置、双向交替起泡

快速排序（不稳定）

  • 分化交换排序/分治法
  • 指定枢轴/支点/基准记录r[p]（通常为第一个记录），通过一趟排序将其放在正确的位置上，它把代排记录分割为独立的两部分，使得左边记录的关键字<=r[p].key<=右边记录的关键字
  • 对左右两部分记录序列重复上述过程，依次类推，直到子序列中只剩下一个记录或不含记录为止
  • 时间复杂度：最坏：O(n2)，最好：O(nlog2n)
  • 辅助空间复杂度：最好O(log2n)，最坏O(n)
  • 算法改进：三者取中、随机产生

选择排序

简单选择排序（不稳定）

  • 选择、交换
  • 总的比较次数与记录排列的初始状态无关C=n*(n-1)/2
  • 移动次数Mmax=3(n-1)，Mmin=0
  • 平均时间复杂度O(n2)，辅助空间复杂度O(1)

树形选择排序（稳定）

  • 时间复杂度O(nlog2n)
  • 缺陷：辅助空间较多，需与“最大值”进行多余的比较

堆排序（不稳定）

  • 堆的调整（筛选）
  • 建堆（从后往前）
  • 时间复杂度O(nlog2n)
  • 空间复杂度：辅助空间O(1)

归并排序

两路归并排序（稳定）

  • 时间复杂度O(nlog2n)
  • 空间复杂度O(n)
  • 很少用于内部排序

单链表的两路归并排序（稳定）

  • 空间复杂度O(1)

自然两路归并排序（不稳定）

  • 以游程（自然的有序段）作为自序列进行归并
  • 前半段寻找正序游程，后半段寻找逆序游程，前半段与后半段对应归并->不稳定

基数排序（稳定）

  • 通过分配与收集过程来实现排序
  • 时间复杂度可以突破给予关键字比较一类方法的下界O(nlog2n)，达到O(n)
  • 最高位优先法（MSD）:分摊->分摊->收集->收集
  • 最低位优先法（LSD）：分摊->收集->分摊->收集
  • LSD：分摊后形成队列，把所有队列头尾相连地收集起来
  • 辅助空间：O(n+rd)

快速排序是目前基于比较的内部排序中最好的方法
