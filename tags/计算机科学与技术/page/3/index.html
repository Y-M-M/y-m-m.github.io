  • leetcode 1870 准时到达的列车的最小时速

    leetcode 1870

    题目

    • 给你一个浮点数 hour ，表示你到达办公室可用的总通勤时间。要到达办公室，你必须按给定次序乘坐 n 趟列车。另给你一个长度为 n 的整数数组 dist ，其中 dist[i] 表示第 i 趟列车的行驶距离（单位是千米）。

    • 每趟列车均只能在整点发车，所以你可能需要在两趟列车之间等待一段时间。

    例如，第 1 趟列车需要 1.5 小时，那你必须再等待 0.5 小时，搭乘在第 2 小时发车的第 2 趟列车。
    返回能满足你准时到达办公室所要求全部列车的 最小正整数 时速（单位：千米每小时），如果无法准时到达，则返回 -1 。

    生成的测试用例保证答案不超过 107 ，且 hour 的 小数点后最多存在两位数字 。

    例子

    • 输入：dist = [1,3,2], hour = 6
    • 输出：1
    • 解释：速度为 1 时：
      1. 第 1 趟列车运行需要 1/1 = 1 小时。
      2. 由于是在整数时间到达，可以立即换乘在第 1 小时发车的列车。第 2 趟列车运行需要 3/1 = 3 小时。
      3. 由于是在整数时间到达，可以立即换乘在第 4 小时发车的列车。第 3 趟列车运行需要 2/1 = 2 小时。
      4. 你将会恰好在第 6 小时到达。

    提示

    • n == dist.length
    • 1 <= n <= 105
    • 1 <= dist[i] <= 105
    • 1 <= hour <= 109
    • hours 中，小数点后最多存在两位数字

    题解

    方法一

    class Solution {
public:
    int minSpeedOnTime(vector<int>& dist, double hour) {
        if (dist.size() - 1 > hour) return -1;
        long long distance = 0;
        for (int i = 0; i < dist.size(); i++) {
            distance += dist[i];
        }
        long long pre_speed = (int)ceil(distance / hour);
        long long speed = pre_speed;
        while (1) {
            double all_time = 0;
            for (int i = 0; i < dist.size() - 1; i++) {
                long long now_dist = dist[i];
                while (now_dist > 0) {
                    now_dist -= speed;
                    all_time++;
                }
            }
            all_time += 1.0 * dist.back() / speed;
            if (all_time <= hour) break;
            else speed = speed + 1.0;
        }
        return speed;
    }
};

    class Solution {
public:
    int minSpeedOnTime(vector<int>& dist, double hour) {
        int n = dist.size();
        // 将hour乘100，再通过llround函数四舍五入为整型
        long long hr = llround(hour * 100);
        // hour > n-1 才能满足条件
        if (hr <= (n - 1) * 100) {
            return -1;
        }

        // 二分查找合适的速度
        int l = 1, r = 1e7;
        while (l < r) {
            int mid = l + (r - l) / 2;
            // 计算速度为mid的时候所需要的总时间
            long long t = 0;
            // 前n - 1段中第i段贡献的时间
            for (int i = 0; i < n - 1; i++) {
                t += (dist[i] - 1) / mid + 1;
            }
            // 最后一段贡献的时间
            t *= mid;
            t += dist[n - 1];
            if (t * 100 <= hr * mid) { // 将分数比较转化为整数比较
                r = mid;
            }
            else {
                l = mid + 1;
            }
        }
        return l;
    }
};

    int mincostTickets(vector<int>& days, vector<int>& costs) {
        vector<bool> is_travel(days.back() + 1);
        for (int i = 0; i <= days.back(); i++) {
            is_travel[i] = 0;
        }
        for (int i = 0; i < days.size(); i++) {
            is_travel[days[i]] = 1;
        }
        vector<int> all_money(days.back() + 1);
        all_money[0] = 0;
        for (int i = 1; i <= days.back(); i++) {
            if (is_travel[i]) {
                if (i >= 7) {
                    all_money[i] = min(all_money[i - 1] + costs[0], all_money[i - 7] + costs[1]);
                    if (i >= 30) {
                        all_money[i] = min(all_money[i], all_money[i - 30] + costs[2]);
                    } else {
                        all_money[i] = min(all_money[i], costs[2]);
                    }
                } else {
                    all_money[i] = min(all_money[i - 1] + costs[0], costs[1]);
                    all_money[i] = min(all_money[i], costs[2]);
                }
            }
            else all_money[i] = all_money[i - 1];
        }
        return all_money[days.back()];
    }
};

  • C++ STL中heap函数的用法

    C++ STL中heap函数的用法

    什么是heap？

    C++STL中提供的一种构建和维护堆的函数接口

    包括哪些函数？

    • make_heap(_First, _Last, _Comp)
      • 将区间内的元素构建成堆
      • _First: 需要构建堆的第一个元素的迭代器；_Last: 需要构建堆的最后一个元素的迭代器
      • _Comp 构建方式
        • less() 大顶堆，默认是大顶堆
        • greater() 小顶堆
        • 需要#include
    • push_heap(_First, _Last, _Comp)
      • 新添加一个元素在末尾(vector的end处)，然后重新调整堆序
      • 第三个参数应该与make_heap保持一致
      • 在使用push_heap之前，应确保已经用push_back()将数据存入到数组中
    • pop_heap(_First, _Last, _Comp)
      • 对heap取出下一个元素，把堆顶元素取出来，放至区间末尾，其余元素进行调整，重新构成堆
      • 第三个参数应与make_heap保持一致
      • 如果要弹出原先的堆顶元素，应再使用pop_back()
    • sort_heap(_First, _Last)
      • 将heap转化为一个已排序群集
      • 将堆进行排序，排序后序列将失去堆的特性，小顶堆所得的群集数字从小到大排列，大顶堆所得的群集从大到小排列
    • is_heap()
      • 测试范围内的元素是否为一个二叉堆
    • is_heap_until()
      • 返回第一个破坏二插堆结构的元素的迭代器

    内部是如何实现的？

    堆排序

    class SeatManager {
public:
    SeatManager(int n) {
        for (int i = 1; i <= n; i++) {
            seat.push_back(i); // 为数组增加新的元素
        }
        make_heap(seat.begin(), seat.end(), greater<int>()); // 构建小顶堆
    }
    
    int reserve() {
        pop_heap(seat.begin(), seat.end(), greater<int>()); 
        // 将小顶堆的堆顶元素弹出来，放到数组最后，调整其他元素构成新的堆
        int ans = seat.back();
        seat.pop_back();
        return ans;
    }
    
    void unreserve(int seatNumber) {
        seat.push_back(seatNumber);
        push_heap(seat.begin(), seat.end(), greater<int>());
    }
private:
    vector<int> seat;
};

/**
 * Your SeatManager object will be instantiated and called as such:
 * SeatManager* obj = new SeatManager(n);
 * int param_1 = obj->reserve();
 * obj->unreserve(seatNumber);

    class Solution {
public:
    int takeCharacters(string s, int k) {
        int len = s.length();
        vector<int> cnt(3);
        for (int i = 0; i < len; i++) {
            cnt[s[i] - 'a']++;
        }
        int min_len = len;
        if (cnt[0] < k || cnt[1] < k || cnt[2] < k)
            return -1;
        if (k == 0) return 0;
        int l = 0;
        for (int r = 0; r < len; r++) {
            // cout << "r = " << r << endl;
            cnt[s[r] - 'a']--;
            // cout << cnt[s[r] - 'a'] << endl;
            while ((cnt[0] < k || cnt[1] < k || cnt[2] < k) && l <= r) {
                // cout << "l = " << l << endl;
                cnt[s[l] - 'a']++;
                l++;
            }
            if (cnt[0] >= k && cnt[1] >= k && cnt[2] >= k) {
                // cout << "l = " << l << ", r = " << r << endl;
                min_len = min(min_len, len-r+l-1);
            }
        } 
        return min_len;
    }
};
    • 提交结果

    • 分析
      • 思路
        • 设置左右指针，通过移动指针来统计数目
        • 先移动右指针，再移动左指针，左指针不超过右指针
      • 优势
        • 相对于解法一，只需记录左指针以左、右指针以右的字符数目，节省空间开销
        • 遍历方法优化：移动右指针后，每次移动左指针都会判断当前是否满足条件，如果不再满足条件就不再移动，最后左指针只需要移动很小的范围，节省时间
        • 在移动一个边界遍历的过程中，另一个边界可从之前的位置开始移动，无需重新开始

    形式语言

    什么是形式语言

    • 形式化描述的字母表上的字符串的集合
    • 字母表：字符的有限集合
    • 字符串：字母表中的字符构成的有限序列

    为什么用形式语言

    • 自然语言：人们平时说话时所使用的一种语言
    • 形式语言：通过人们公认的符号，表达方式所描述的一种语言，是一种通用语言，没有国籍之分

    自动机

    什么是自动机

    • 具有离散输入输出的数学模型
    • 接受一定的输入，执行一定的动作，产生一定的结果。使用状态迁移描述整个工作过程
    • 状态：一个标识，能区分自动机在不同时刻的状况。有限状态系统具有任意有限数目的“内部状态‘
    • 自动机的本质：根据状态、输入和规则决定下一个状态
    • 状态+输入+规则->状态迁移

    分类

    • 有限状态自动机：由一个带有读头的有限控制器和一条写有字符的输入带组成
    • 下推自动机：由一条输入带，一个有限控制器和一个下推栈组成
    • 基本图灵机：由一个具有读写头的有限控制器和一条无限带组成

    形式语言与自动机的关系

    密切相关

    语言的定义与运算

    语言的一些术语

    • 字母表：字符的有限集合，记为T
    • 字符串：由字母表T中的字符构成的序列称字母表T上的字符串（句子）
      • 常记为t,u,v,w,x,y,z
      • 常用a,b,c,d来表示单个字符
      • 常用an表示n个a的字符串

    字母表

    • 概念 形式符号的集合
    • 记号 T
    • 举例
      • 英文字母表
      • 英文标点符号表
      • 汉字表

    字符串

    • 字母表T上的一个字符串（串、字），为T中的字符构成的一个有限序列，空串，用ε来表示
    • 绝对值符号，表示字符串的长度
    • a的i次幂，表示含有i个a的字符串

    连接concatenation

    • 连接运算的性质
      • 结合律
      • 空串是单位元
      • 连接后的长度等于各串的长度之和

    其他运算

    • 取头字符
    • 取尾部
    • 字串匹配
    • 前缀、后缀、子串
    • 空串是任意串的前缀、后缀及子串
    • 字符串的逆：将字符串倒置
    • 空串的逆还是它本身

    字母表的幂运算

    • （1）T0 空串 Tn长度为n的字符串
    • （2）设x在Tn-1中，那么ax在Tn中
    • （3）Tn中的元素只能由（1）和（2）构成
    • *闭包 字母表T中的字符所构成的任意长度字符串的集合
    • +闭包 没有空串的*闭包

    • 英文单词集
    • C语言程序集
    • 汉语成语集
    • 化学分子式集

    运算

    • 并、交、补

    • 语言的幂
      • L0，仅含有空串的集合

    • 若语言L是无限集合，用其他方法

      • 机器识别系统（自动机），当一个字符串能被一个语言识别系统接受，则这个字符串是该语言的一个句子，否则不属于该语言

    Chomsky文法体系

    BNF

    • <标识符>::=<字母>
    • <标识符>::=<标识符><字母>

    • <数字>::=0|1|……9

    • I->ID
    • L->a|b|……|z

    • S 起始符 且S ∈N。 I

    • 从å0到å0的直接推导，是长度为0的推导序列

    • 句子是由终结符组成的字符串

    文法产生的语言

    Chomsky文法体系分类

    • 对生成式的形式作规定，分为0型、1型、2型、3型文法
    • 0型文法：无限制文法
    • 1型文法：包括上下文有关文法(GSG)->上下文有关语言(GSL)->线性有界自动机(LBA) 把计算限制在仅仅包含输入的那一段带上的图灵机，可作为上下文有关语言的识别接受器
      • 左部的长度小于右部
      • 不含空串
    • 2型文法：上下文无关文法(CFG)->上下文无关语言(CFL)->下推自动机(PDA)，左部是单个非终结符
    • 3型文法：正则文法->有限自动机
      • 右线性文法A->ω或A->ωB
      • 左线性文法A->ω或A->Bω
    • 3型文法属于2型文法，不含A->ε的2型、3型文法属于1型，1型、2型、3型均属于0型

    第三章 有限自动机与右线性文法

    有限自动机

    有限状态系统

    • 状态：可以将事物区分开的一种标识
    • 具有离散状态的系统：离散状态系统的状态数是有限的
    • 具有连续状态的系统，有无限个状态
    • 有限状态系统必然是离散状态系统

    有限自动机的概念

    • 具有离散输入输出的系统的一种模型（可以没有输出，比较特殊的也可以没有输入）
    • 有限的状态
    • 状态+输入->状态转移
    • 每次转换的后继状态都唯->DFA确定的
    • 每次转换后继状态不唯一->NFA不确定的

    FA的模型

    • FA可以理解成一个控制器，它读一条输入带上的字符
    • 控制器包括有限状态
    • 从左到右依次读取字符
    • 状态+激励->状态转移

    有限自动机

    • 有限状态集
    • 有限输入符号集
    • 转移函数
    • 一个开始状态
    • 一个终态集合，用双圈表示，可以不止一个

    DFA

    DFA的形式定义

    • DFA是一个五元组 M = (Q, T, δ, q0, F)
      • Q:有限的状态集合
      • T:有限的输入字母表
      • δ:转换函数
      • q0:初始状态
      • F:终止状态集
    • 状态转移图
    • 状态转移表
      • 最左侧为状态
      • 起始状态加箭头，终止状态加*

    扩展转移函数适合于输入字符串

    • δ’函数
      • δ’: Q * T* -> Q

    DFA接受的语言

    • DFA接受的字符串：输入结束后使DFA的状态到达终止状态的字符串
    • DFA接受的语言：被DFA接受的字符串的集合就是DFA接受的语言

    格局

    • 描述有限自动机在某一时刻的工作状态，可以用两个信息表示：当前状态q，待输入字符串w。两者构成一个顺时描述，用(q, w)表示
    • 初始格局：(q0, w)
    • 终止格局：(q, ε)，q属于F
    • 格局数量是无限的
    • 有限状态自动机是无记忆的

    设计有限自动机

    不确定的有限自动机(NFA)

    • 一个输入可以有多个转移，到达不同的状态，则称为不确定的有限自动机
    • δ: Q * T -> 2Q, 其余与DFA相同
